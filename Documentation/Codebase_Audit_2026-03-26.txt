2. SECURITY ISSUES
================================================================================

[HIGH] Template XSS with nofilter (travel_core):
  - travel_bookings/view.tpl:28    {$booking.provider_display.status_label nofilter}
  - travel_bookings/manage.tpl:220 Same pattern
  Risk: Provider-controlled HTML could inject malicious content.
  Fix: Use |escape:'html' or ensure providers return pre-escaped data.

[MEDIUM] Error suppression operators (sphinx_holidays):
  - controllers/backend/sphinx_holidays.php:38,61,194 -> @set_time_limit(0)
  Fix: Use conditional: if (function_exists('set_time_limit')) { set_time_limit(0); }

[GOOD] No SQL injection, proper auth, CSRF protection via CS-Cart dispatcher.
