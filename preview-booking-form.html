Style Differences: New vs Current

Key architectural change: The form container itself becomes the golden element. Instead of a white box with a golden border, the container is gold-colored and each field sits inside as a separate white rounded box. The golden gap: 4px between boxes creates unified golden dividers.
Property Current New
Container background #ffffff (white) #FFB703 (gold)
Container border 3px solid #FFB703 4px solid #FFB703
Container height auto (min-height: 40px inner) 64px fixed
Field dividers border-right: 1px solid #e0e0e0 gap: 4px (gold background shows through)
Field boxes No individual background White #fff with border-radius: 4px
Icon color #6b6b6b #4a4a4a (slightly darker)
Label font 14px / 500 weight 15px / 400 weight
Arrow color #6b6b6b #a0a0a0 (lighter)
Button color #0071E3 #0071c2 (Booking blue)
Button font 15px / 600 weight 18px / 700 weight (bolder)
Button border-radius 8px 4px (matches field boxes)
Box shadow none 0 4px 12px rgba(0,0,0,0.08)

Product Mode

Shown on hotel/product pages. Has "Availability" header above the form. Fields: date, guests, Search button.

Search Mode

Shown on search results page. Has "Change search" header with refresh icon. Button text cycles: Search → Change search → Apply changes.

Homepage Mode

Includes location text input as the first field. All four fields (location, date, guests, button) in one golden frame.

Features Preserved (No Changes)

All existing functionality remains identical:
  • Calendar popup (two-month side-by-side, per-date pricing, range highlighting)
  • Guest/Room picker popup (multi-room, adults/children, child age selectors)
  • Smart age anchor (clickable missing-age warnings)
  • SVG icons: CalendarIcon, GuestIcon, ChevronDown, ChevronLeft, ChevronRight, TrashIcon, RefreshIcon
  • Validation messages (dates required, child ages required)
  • AJAX search on results page (no page reload)
  • Button state machine: Search → Change search → Apply changes
  • Bilingual support (English & Romanian)
  • Responsive layout (vertical stacking on mobile)
  • CSS variables for theme overrides
